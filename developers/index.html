API Overview

The Ripplesk API is a RESTful HTTP API for building custom integrations, automating conversations, managing contacts, and controlling your support platform programmatically. All requests use HTTPS and return JSON.

Base URL:

https://api.ripplesk.com/v1
🔐 Secure

TLS 1.3 only. API keys scoped by permission.

⚡ Rate Limits

1,000 req/min on Pro. 10,000 on Enterprise.

📦 Versioned

Current: v1. Breaking changes are versioned.

Quick Start

Get your first conversation created in under 5 minutes. Pick your language:

cURL
JavaScript
Python
PHP
# Create a conversation via cURL curl -X POST https://api.ripplesk.com/v1/conversations \ -H "Authorization: Bearer rsk_live_YOUR_KEY" \ -H "Content-Type: application/json" \ -d '{ "channel": "web", "contact": { "name": "Jane Doe", "email": "jane@example.com" }, "message": "Hello! How can I help you today?" }'
// npm install @ripplesk/sdk import Ripplesk from '@ripplesk/sdk'; const client = new Ripplesk({ apiKey: 'rsk_live_YOUR_KEY' }); const conversation = await client.conversations.create({ channel: 'web', contact: { name: 'Jane Doe', email: 'jane@example.com' }, message: 'Hello! How can I help you today?' }); console.log(`Created: ${conversation.id}`);
# pip install ripplesk import ripplesk client = ripplesk.Client(api_key="rsk_live_YOUR_KEY") conversation = client.conversations.create( channel="web", contact={"name": "Jane Doe", "email": "jane@example.com"}, message="Hello! How can I help you today?" ) print(f"Created: {conversation['id']}")
// composer require ripplesk/ripplesk-php use Ripplesk\Client; $client = new Client('rsk_live_YOUR_KEY'); $conv = $client->conversations->create([ 'channel' => 'web', 'contact' => ['name' => 'Jane Doe', 'email' => 'jane@example.com'], 'message' => 'Hello!' ]); echo "Created: " . $conv['id'];

Authentication

Pass your API key in the Authorization header on every request:

Authorization: Bearer rsk_live_xxxxxxxxxxxxxxxxxxxxxxxx
Key TypePrefixUse
Live keyrsk_live_Production
Test keyrsk_test_Development — no real messages sent
Restricted keyrsk_res_Scoped to specific resources
⚠️

Never expose API keys in client-side code or public repos. Use environment variables and rotate regularly.

Conversations

The central resource — each conversation belongs to a contact, has a channel, and contains a message thread.

GET/v1/conversationsList conversations
POST/v1/conversationsCreate a conversation
GET/v1/conversations/:idRetrieve one conversation
PUT/v1/conversations/:idUpdate a conversation
DELETE/v1/conversations/:idDelete a conversation

Response example

{ "id": "conv_9f3kp2", "channel": "whatsapp", "status": "open", "contact": { "id": "cnt_abc123", "name": "Jane Doe", "email": "jane@example.com" }, "labels": ["billing", "urgent"], "created_at": "2025-06-14T09:23:11Z" }

Messages

Send and retrieve messages within a conversation. Sender can be agent, bot, or contact.

GET/v1/conversations/:id/messagesList messages
POST/v1/conversations/:id/messagesSend a message
ParameterTypeRequiredDescription
contentstringrequiredMessage text
typestringrequiredoutgoing | note | template
senderobjectrequiredSender type and id
attachmentsarrayoptionalFile attachment objects

Contacts

Manage your customers. Each contact can have multiple conversations across channels.

GET/v1/contactsList contacts
POST/v1/contactsCreate a contact
GET/v1/contacts/:idRetrieve a contact
PUT/v1/contacts/:idUpdate a contact
DELETE/v1/contacts/:idDelete a contact

Webhooks

Subscribe to real-time events. Ripplesk POST requests to your URL whenever something happens.

conversation.created
conversation.resolved
message.created
message.read
contact.created
bot.handoff

Webhook payload example

{ "event": "message.created", "timestamp": "2025-06-14T09:23:11Z", "account_id": "acc_abc123", "data": { "message": { "id": "msg_99xyz", "content": "Where is my order?", "type": "incoming", "conversation_id": "conv_9f3kp2", "sender": { "type": "contact", "id": "cnt_abc123" }, "created_at": "2025-06-14T09:23:11Z" } } }
